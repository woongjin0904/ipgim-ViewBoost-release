# **🛠️ 저장소 완전 초기화 및 클린 재배포 가이드**

이 가이드는 GitHub LFS 용량 문제를 해결하고, 꼬인 이력을 모두 제거하여 **GitHub Pages 기반의 자동 업데이트 시스템**을 처음부터 다시 구축하는 단계를 설명합니다.

## **1단계: 원격 저장소 초기화 (GitHub 서버 비우기)**

GitHub 서버에 남아있는 과거의 흔적을 삭제하여 클라이언트와의 충돌을 방지합니다.

### **1\. 모든 GitHub Releases 삭제 (CLI 사용)**

터미널에서 아래 명령어를 실행하여 기존 릴리즈를 일괄 삭제합니다.

\# 릴리즈 목록을 가져와서 하나씩 삭제 (gh CLI 로그인 상태여야 함)  
gh release list \--repo woongjin0904/ipgim-ViewBoost-release \--limit 100 | cut \-f 1 | xargs \-n 1 gh release delete \--repo woongjin0904/ipgim-ViewBoost-release \--yes

### **2\. 모든 Git 태그 삭제 (로컬 및 원격)**

버전 정보가 꼬이지 않도록 모든 태그 기록을 지웁니다.

\# 로컬 태그 전체 삭제  
git tag \-l | xargs git tag \-d

\# 원격 저장소 태그 전체 삭제  
git tag \-l | xargs \-n 1 bash \-c 'git push origin :refs/tags/$0'

### **3\. GitHub Pages(gh-pages) 브랜치 삭제**

업데이트 파일이 호스팅되는 브랜치를 날려버립니다. (이미 없다면 에러가 나지만 무시하세요.)

git push origin \--delete gh-pages

## **2단계: 로컬 작업 환경 초기화 (내 컴퓨터 비우기)**

이전 빌드 파편들이 새 패키지에 포함되지 않도록 싹 정리합니다.

### **1\. 배포용 임시 폴더 및 빌드 결과물 삭제**

\# 로컬 폴더 정밀 소거  
rm \-rf vpk-out/  
rm \-rf temp-deploy/  
rm \-rf dist/  
rm \-rf dist-client/

### **2\. 로컬 Git 이력 초기화 (선택 사항이지만 권장)**

LFS 기록이 너무 무거워 pull/push가 안 된다면 .git 폴더 자체를 새로 만드는 것이 가장 확실합니다.

rm \-rf .git  
git init  
git remote add origin \[https://github.com/woongjin0904/ipgim-ViewBoost-release.git\](https://github.com/woongjin0904/ipgim-ViewBoost-release.git)  
git checkout \-b main

## **3단계: 설정 점검 (Configuration)**

새로운 시작을 위해 프로젝트 설정을 확인합니다.

1. **package.json 버전**: 버전을 "1.0.0"으로 되돌립니다.  
2. **publish.js 확인**: 상단의 RELEASE\_REPO\_URL이 실제 배포 저장소 주소와 맞는지 확인합니다.  
3. **.env 파일 확인**: 루트 폴더에 .env가 있고, 인증서 비밀번호와 복호화 키가 정확하게 입력되어 있는지 확인합니다.

## **4단계: 첫 번째 'Clean' 배포 실행**

이제 모든 준비가 끝났습니다. 자동 배포 스크립트를 실행합니다.

node publish.js

**이 스크립트가 자동으로 수행하는 작업:**

* temp-deploy 폴더 생성 및 초기화  
* Velopack을 이용한 1.0.0 빌드 (RELEASES 신규 생성)  
* .nupkg와 RELEASES 파일만 필터링하여 복사  
* GitHub gh-pages 브랜치에 강제 푸시(Force Push)

## **5단계: 배포 결과 확인**

배포가 성공적으로 끝났다면, 브라우저를 열어 업데이트 서버가 정상 작동하는지 확인합니다.

🔗 **확인 주소**: [https://woongjin0904.github.io/ipgim-ViewBoost-release/RELEASES](https://woongjin0904.github.io/ipgim-ViewBoost-release/RELEASES)

위 주소에서 1.0.0 관련 텍스트 한 줄이 보인다면 **완벽하게 성공**한 것입니다. 이제부터는 버전만 올리고 node publish.js를 실행하기만 하면 업데이트가 자동으로 누적됩니다.

### **💡 주의사항**

* **LFS 절대 금지**: 앞으로 main 브랜치에 큰 파일을 직접 올리지 마세요.  
* **보안**: .env와 .pfx 파일이 실수로 GitHub에 업로드되지 않도록 .gitignore 설정을 항상 유지하세요.
